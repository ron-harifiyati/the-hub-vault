Phase 1 — 10:00 to 10:45

The Problem We're Solving

⚠️ The iPad Wipe Problem At the end of every cohort, hub iPads are reset. Every presentation, every project, every piece of student work — gone. Unless it's somewhere else.

❌ Without GitHub

  • Work lives only on the iPad
  • iPad gets wiped — everything gone
  • Student has no proof of their progress
  • "I did the work" — but can't show it
  • Sponsor can't see what their money funded

✅ With GitHub

  • Work is in the cloud, permanent
  • iPad wipe? Doesn't matter
  • Student has a timestamped public portfolio
  • Every upload is a receipt with a date and time
  • Sponsor can see everything, from anywhere
The Blunt Truth If it's not on GitHub, it didn't happen. That's the rule. That's what we're here to fix today.

Phase 1 — What is GitHub?

Three Things GitHub Does For Us

Forget the technical definition. Here is what it actually means for your hub.

📁

The Folder

A cloud home for project files. Like Google Drive — but built for professional creators. Everything in one place, accessible from any device.

⏳

The Time Machine

It remembers every version of every file. Delete a slide on Monday — GitHub has it from Sunday. Nothing is ever truly lost.

🛂

The Digital Passport

How MCRI and employers see a student before they meet them. A filled GitHub profile says: "I showed up. I did the work. Here is the proof."

Phase 2 — 10:45 to 11:30

The iPad-to-GitHub Pipeline

Four steps. This is the workflow your students will repeat every week.

  • 1

    Keynote — Create the Work

    Build or open a presentation. For this demo: a single slide saying "Hello from [Your Hub]."

  • 2

    Export — Convert to PDF

    Tap ··· → Export → PDF → Good quality. Critical: Keynote files only open in Keynote. PDFs open everywhere.

  • 3

    Save — Store in Files App

    Tap "Save to Files." Give it a clear name: Hello-HarareHub-YourName.pdf

  • 4

    Upload — Push to GitHub

    Open Safari → hub repo → Add file → Upload files → select the PDF → write commit message → Commit.

Phase 2 — Step 1

Keynote: Create Your Slide

🖥️ Mirror Your iPad Now Everyone watches you first. Then they copy. Don't let them race ahead.

What to Create

  • Open Keynote on your iPad
  • Create a new presentation — any theme
  • Title slide only: "Hello from [Your Hub Name]"
  • Add your name as subtitle
  • That's it — one slide is enough for this demo

Why Not Just Upload the Keynote?

A .key file only opens in Apple Keynote. A PDF opens in:

  • Any web browser
  • Any phone or tablet
  • A sponsor's Windows laptop
  • GitHub's built-in viewer — no download needed

Phase 2 — Step 2

Export as PDF from Keynote

  • 1

    Tap the three dots (···)

    Top-right corner of Keynote — the overflow menu

  • 2

    Tap "Export"

    Then select PDF from the format options

  • 3

    Choose Quality: GOOD — not Best

    Best = 50MB+ file. Good = under 5MB. GitHub's browser upload limit is 25MB. Always choose Good.

  • 4

    Tap Export again to confirm

    The PDF is now being generated. A share sheet will appear next.

⚠️ Quality Selector — Say This Out Loud "Tap Good. Not Best. Good. If you tap Best, your file will be too large to upload and you will have to start the export again."

Phase 2 — Step 3

Save to the Files App

  • 1

    From the share sheet, tap "Save to Files"

    The share sheet appears after you complete the export. Look for the Files icon.

  • 2

    Choose your save location

    Tap On My iPad or iCloud Drive. Either works — just remember which one.

  • 3

    Rename the file before saving

    Use a clear, professional name: Hello-HarareHub-TendaiMoyo.pdf — Hub name + Your name. No spaces.

  • 4

    Tap Save

    Your PDF is now in the Files app. Open Files to confirm it's there before switching to Safari.

💡 Quick Check Before moving to Step 4, open the Files app and find your PDF. Don't proceed until you can see it. One minute of confirmation saves five minutes of frustration.

Phase 2 — Step 4

Upload to GitHub in Safari

  • 1

    Open Safari → navigate to the Collaboration Repo

    Use the URL posted on screen or in the group chat. Make sure you're logged in.

  • 2

    Tap "Add file" → "Upload files"

    This button is near the top of the file list. If you don't see it, you may not be logged in.

  • 3

    Tap "choose your files" → find your PDF

    Navigate: On My iPad → Keynote → select your file. Or use Recents at the top of the Files picker.

  • 4

    Write a commit message → tap "Commit changes"

    Message: "Hello from [Hub Name] — [Your Name]". Then tap the green Commit button.

Phase 3 — File Management

Editing, Renaming & Deleting Files

✏️

Editing a File

Click the file → click the pencil icon (✏️). Works best for text files like README. For PDFs, just upload a new version.

🗑️

Deleting a File

Click the file → click the trash icon. Important: the file is removed from the view but stays in the commit history. Nothing is truly erased.

🔒

Public vs. Private

Public: anyone can see it — no login. Use for hub repos so sponsors can access easily. Private: only you and invited people. Use for personal drafts.

⚠️ Our Rule Hub repositories should always be Public. Sponsors should never need a GitHub account to see the work we are doing.

Phase 4 — 14:00 to 14:45

Scenario: What Would You Do?

🎭 Scenario A student tells you: "I finished my Blue's Adventure presentation but the iPad was wiped at the end of last cohort and my Keynote file is gone."

Without GitHub

There is nothing you can do. The work is gone. The student has no proof, no portfolio, no record. The cohort ends and they have nothing to show the MCRI.

With GitHub

You check their repository. If they committed the PDF — it's there, timestamped, permanent. If they didn't commit it — this is the teachable moment. This is the lesson they will never forget.

💬 The Question to Ask Students "Did you upload it to GitHub?" That's it. Two outcomes: yes (they're safe) or no (they learn the rule the hard way, once).

Phase 4 — Weekly Routine

Integrating GitHub Into Every Week

The workflow only works if it becomes a habit. Here's how to build that habit at your hub.

📅

For Students

Set a fixed time: "Every Friday at 2 PM, we upload our Keynotes." Consistency is more important than perfection. One upload per week, every week, builds an undeniable record.

📊

For Facilitators

Every Friday: write the week's report, export as PDF, upload to the hub repo, share the repo URL with sponsors. Replace the email attachment habit with the link habit.

👀

Monitoring

At the start of each session, pull up 2–3 student commit histories on the big screen. Showing students their own record of growth is the strongest motivational tool you have.

💡 Make It a Ritual Name it. "GitHub Friday." "Upload Day." Whatever sticks. Rituals that have names get done. Rituals that are vague get skipped.
