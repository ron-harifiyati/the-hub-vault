Phase 1 — 10:00 to 10:45

The Problem We're Solving

⚠️ The iPad Wipe Problem At the end of every cohort, hub iPads are reset. Every presentation, every project, every piece of student work — gone. Unless it's somewhere else.

❌ Without GitHub

  • iPad gets wiped — everything gone
  • No proof. No portfolio. No record.
  • Sponsors can't see what their money funded

✅ With GitHub

  • iPad wipe? Doesn't matter
  • Every upload is timestamped — permanent proof
  • Sponsors see everything, from anywhere, no login
The Blunt Truth If it's not on GitHub, it didn't happen. That's the rule. That's what we're here to fix today.

Phase 2 — 10:45 to 11:30

The iPad-to-GitHub Pipeline

Four steps. This is the workflow your students will repeat every week.

1

Keynote — Create

One slide: "Hello from [Your Hub]." Any theme, any size.

2

Export — PDF

Tap ··· → Export → PDF → Good quality. PDFs open everywhere — Keynote files don't.

3

Save — Files App

Save to Files with a clear name: Hello-HarareHub-YourName.pdf

4

Upload — GitHub

Safari → repo → Add file → Upload → select PDF → commit message → Commit.

Phase 2 — Step 1

Keynote: Create Your Slide

🖥️ Mirror Your iPad Now Everyone watches you first. Then they copy. Don't let them race ahead.

What to Create

  • Open Keynote on your iPad
  • Create a new presentation — any theme
  • Title slide only: "Hello from [Your Hub Name]"
  • Add your name as subtitle
  • That's it — one slide is enough for this demo

Why Not Just Upload the Keynote?

A .key file only opens in Apple Keynote. A PDF opens in:

  • Any web browser
  • Any phone or tablet
  • A sponsor's Windows laptop
  • GitHub's built-in viewer — no download needed

Phase 2 — Step 2

Export as PDF from Keynote

1

Tap the three dots (···)

Top-right corner of Keynote — the overflow menu.

2

Tap "Export" → PDF

Select PDF from the format options.

3

Choose Quality: GOOD

Best = 50MB+. Good = under 5MB. GitHub limit is 25MB. Always choose Good.

4

Tap Export to confirm

PDF is generated. A share sheet appears next.

⚠️ Say this out loud "Good. Not Best. Good. Best = file too large to upload."

Phase 2 — Step 3

Save to the Files App

1

Tap "Save to Files"

From the share sheet after export. Look for the Files icon.

2

Choose save location

On My iPad or iCloud Drive. Either works — remember which.

3

Rename the file

Hello-HarareHub-TendaiMoyo.pdf — Hub + Name. No spaces.

4

Tap Save — then verify

Open Files app and confirm your PDF is there before switching to Safari.

💡 Quick Check Don't move to Step 4 until you can see your PDF in the Files app.

Phase 2 — Step 4

Upload to GitHub in Safari

1

Open Safari → Collaboration Repo

Use the URL on screen or in the group chat. Confirm you're logged in.

2

Tap "Add file" → "Upload files"

Near the top of the file list. Don't see it? You're not logged in.

3

Find your PDF

On My iPad → Keynote → select file. Or use Recents at the top of the picker.

4

Write commit message → Commit

"Hello from [Hub Name] — [Your Name]" → tap the green Commit button.

Phase 4 — 14:00 to 14:45

Scenario: What Would You Do?

🎭 Scenario A student tells you: "I finished my Blue's Adventure presentation but the iPad was wiped at the end of last cohort and my Keynote file is gone."

Without GitHub

Nothing you can do. The work is gone. No proof, no portfolio, no record.

With GitHub

Check the repo. Committed? It's there — timestamped, permanent. Not committed? That's the lesson they never forget.

💬 The Question to Ask Students "Did you upload it to GitHub?" That's it. Two outcomes: yes (they're safe) or no (they learn the rule the hard way, once).
