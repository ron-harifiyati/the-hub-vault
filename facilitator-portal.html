    Step One

    Hub Organization — Setting the Standard

    Every hub in Zimbabwe should have its own GitHub repository. This is the hub's official home on GitHub — where all reports, facilitator updates, and evidence of activity live.

    Repository Naming Convention

    Consistency matters. Use this naming format across all hubs:

    Hub Name Repository Name Example URL
    Mosi-oa-Tunya Hub MosiHub-Cohort-2026 github.com/[org]/MosiHub-Cohort-2026
    Harare Hub HarareHub-Cohort-2026 github.com/[org]/HarareHub-Cohort-2026
    Bulawayo Hub BulawayoHub-Cohort-2026 github.com/[org]/BulawayoHub-Cohort-2026
    Mutare Hub MutareHub-Cohort-2026 github.com/[org]/MutareHub-Cohort-2026
    Gweru Hub GweruHub-Cohort-2026 github.com/[org]/GweruHub-Cohort-2026

    ✅ Make it Public

    Keep the hub repository Public. Sponsors need to see the work without needing a GitHub account or a technical walkthrough. Public means accessible.

    📁 Suggested Folder Structure

    Inside the repo, create folders by uploading files into named paths:

    • reports/ — weekly facilitator PDFs
    • students/ — student work (or link to student repos)
    • photos/ — hub activity images

    Step Two

    The Reporting Cycle — Sponsor Transparency

    The Friday report is the single most important thing you upload each week. It is your proof of activity, your professional communication to sponsors, and your contribution to the MCRI visibility record.

    The Friday Workflow

    1. Write your weekly facilitator report in Keynote or Pages
    2. Export it as a PDF (same process as students — see Student Zone Step 3)
    3. Name it clearly: Week-12-HarareHub-Report.pdf
    4. Upload it to the reports/ folder in your hub repository
    5. Write a commit message: "Week 12 report — 14 students active, Swift Level 3 completed"
    6. Tap Commit changes
    📧 Stop Emailing Attachments Instead of attaching the PDF to an email, send the sponsor the URL of your repository. It shows them all reports in one place, demonstrates you use professional tools, and never fills up anyone's inbox.

    What to Include in Every Report

    • Number of students present that week
    • Topic or module covered
    • Any challenges or support needed
    • A brief highlight (e.g., "Tariro finished her first app prototype")
    • Photos of the session (if possible)

    Step Three

    Monitoring Progress — The Activity Feed

    GitHub gives you a built-in attendance sheet for free. Every time a student uploads their work, it is recorded with a timestamp. These are called Commits — and they are the receipts.

    How to Check a Student's Commits

    1. Go to the student's repository (e.g., github.com/TendaiMoyo/My-Swift-Journey)
    2. Look for the Commits count near the top of the file list
    3. Tap it to open the full commit history
    4. Each entry shows: the commit message, the date, and the time
    [IMAGE: GitHub repository commits tab on iPad Safari — showing a list of commits with student names, descriptions, and timestamps visible]
    📅

    Checking Consistency

    A student who commits every week has a clear record of discipline. A student with no commits for 3 weeks needs a check-in conversation — the evidence is right there.

    🕐

    The Timestamp is the Proof

    GitHub records the exact date and time of every upload. "I did the work" is unprovable. A commit timestamped Tuesday at 2:47 PM is not.

    💡 Facilitator Tip At the start of each session, pull up the student's commit history on a shared screen. Showing students their own timeline of growth is one of the most powerful motivational tools you have.

    Step Four

    Feedback — Using GitHub Issues

    GitHub Issues is a built-in commenting system. Use it to leave structured, professional feedback directly on a student's repository — so they have a written record of their growth.

    How to Leave Feedback via Issues

    1. Go to the student's repository
    2. Tap the Issues tab at the top
    3. Tap New issue
    4. Write a clear title: "Week 4 Feedback — Loops Presentation"
    5. Write your feedback in the body
    6. Tap Submit new issue

    ✅ Good Feedback

    "Great job on the Loops slide — the animation made the concept clear. For next time, try adding a real-world example, like a shopping cart counting items."

    ❌ Unhelpful Feedback

    "Good work. Keep it up."

    This gives the student nothing to act on.

Sponsor & Hire-Up View

The Glass Window

This section is for sponsors, MCRI staff, and anyone who wants to observe hub progress without needing a GitHub account or any technical knowledge.

🔗

How to Access a Hub

Ask your facilitator for the repository URL. It looks like:
github.com/[org]/HarareHub-Cohort-2026

Open that URL in any web browser. No login needed.

📂

What You Will See

A file list showing all uploaded reports, student work folders, and activity photos. Click any PDF file to view it directly in your browser — no download required.

📊

Reading the Activity

The Commits count (visible near the file list) shows total uploads. Click it to see every upload — who, what, and when. This is the live pulse of the hub.

The Impact You are not just funding a workshop. You are watching the birth of a developer ecosystem in real-time. Every commit is a student who showed up, did the work, and left a permanent record of their growth.
