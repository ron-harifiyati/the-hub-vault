Student Zone

Your Digital Journey

Four steps to go from a Keynote on your iPad to a professional GitHub profile that represents who you are and what you can do.

⚠️ Read This First iPads are wiped at the end of every cohort. If your work is only on the iPad, it will be gone. If it is on GitHub, it lives forever. This is not optional — this is how we protect your progress.

The Pipeline

Four Steps to the Vault

Follow these in order. The first time takes 20 minutes. After that, it takes 5.

  • 1

    The Account — Your Professional Identity

    Go to github.com in Safari on your iPad and create a free account. Your username is permanent and public — choose it carefully.

    Choosing your username

    • ✅ TendaiMoyo — professional, identifiable
    • ✅ Tendai-Dev — clean, memorable
    • ❌ coolkid99 — unprofessional
    • ❌ xXxTendaixXx — hard to take seriously

    After signing up, complete your profile:

    • Upload a clear headshot (not a cartoon, not a logo)
    • Write a bio: "Student at [Hub Name] | Aspiring Swift Developer"
    • Add your location: Zimbabwe
    [IMAGE: GitHub profile setup screen on iPad Safari — showing the bio, photo upload, and location fields]
  • 2

    The Repository — Your Project Box

    A repository (repo) is a folder on GitHub that holds all the files for one project. You will create one for your learning journey.

    How to create your first repository

    1. Tap the + icon in the top-right corner of GitHub
    2. Select New repository
    3. Name it: My-Swift-Journey
    4. Set it to Public
    5. Tick "Add a README file"
    6. Tap Create repository
    [IMAGE: GitHub "New Repository" form on iPad Safari — showing the name field set to "My-Swift-Journey" and the Public/README options]
    Write your README Your README is the front cover of your project. Open it and write 2 sentences: what you are learning and why. Example: "Tracking my progress in Swift Learn to Code 1 & 2. I am building towards a career in iOS development at the Matter Career Readiness Institute."
  • 3

    The iPad-to-GitHub Pipeline — The Workflow

    This is the most important step. Your Keynote file cannot be read by everyone — but a PDF can. You must export to PDF every time before uploading.

    ⚠️ Critical: Always Export as PDF A Keynote file only opens in Keynote. A PDF opens in any browser, on any device, anywhere in the world — including in a sponsor's browser. This is the only format that guarantees your work is readable.

    Step A — Export from Keynote

    1. Open your presentation in Keynote
    2. Tap the three dots (···) in the top-right
    3. Tap Export
    4. Choose PDF
    5. Select quality: Good (not Best — keeps file size small)
    6. Tap Export again
    [IMAGE: iPad Keynote Share Sheet — showing the Export option with PDF highlighted and the quality selector set to "Good"]

    Step B — Save to Files

    1. When the export finishes, tap Save to Files
    2. Choose On My iPad or iCloud Drive
    3. Give it a clear name: Swift-Level5-TendaiMoyo.pdf
    4. Tap Save
    [IMAGE: iPad Files app — showing the PDF saved in the "On My iPad" folder with a clear filename]

    Step C — Upload to GitHub

    1. Open Safari and go to your GitHub repository
    2. Tap Add file → Upload files
    3. Tap choose your files
    4. Navigate to your PDF in Files and select it
    5. Scroll down — you will see the Commit section
    [IMAGE: GitHub repository on iPad Safari — showing the "Add file" dropdown and the "Upload files" option]
  • 4

    The Commit — The Save With a Label

    A commit is a permanent save point. It records what you uploaded and when. This timestamp is your receipt — proof that you did the work on that date.

    Writing a good commit message

    Always describe what you did, not just "upload".

    • ✅ "Finished Blue's Adventure Level 5 — Loops chapter"
    • ✅ "Week 3 presentation — Variables and Functions"
    • ❌ "upload"
    • ❌ "stuff"
    • ❌ "asdfjkl"
    [IMAGE: GitHub commit message field on iPad Safari — showing a descriptive commit message typed in and the green "Commit changes" button]
    ✅ You Did It After tapping "Commit changes", your work is in the vault. It is timestamped, permanent, and accessible from anywhere in the world. That is professionalism.
